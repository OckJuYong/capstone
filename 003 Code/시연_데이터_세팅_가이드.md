## 1. 시연 개요

### 앱 컨셉
> **"기존 배달앱은 카테고리로만 검색하지만, MobiSync는 내 맛 취향을 분석해서 나에게 맞는 음식을 추천합니다"**

### 핵심 차별점 3가지
| 순서 | 기능 | 설명 |
|------|------|------|
| 1 | **맛 취향 설정 슬라이더** | 6가지 맛(매운맛/단맛/짠맛/신맛/감칠맛/쓴맛)을 5단계로 설정 |
| 2 | **매칭률 % 표시** | "내 취향 89% 일치" 형태로 추천 결과 제공 |
| 3 | **6축 맛 프로필 차트** | 레스토랑/메뉴별 맛 분석 시각화 |

### BCrypt 비밀번호 생성 방법

Spring Boot에서 비밀번호 암호화:
```java
// 테스트 코드나 DataInitializer에서 실행
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
String encodedPassword = encoder.encode("demo1234");
System.out.println(encodedPassword);
// 출력된 값을 SQL에 사용
```

또는 온라인 BCrypt 생성기 사용:
- https://bcrypt-generator.com/
- 비밀번호: `demo1234` → BCrypt 해시 생성

## 5. 시연 전 체크리스트

### 백엔드 팀 체크
```
□ MariaDB에 위 SQL 스크립트 실행 완료
□ demo@mobisync.com / demo1234 로그인 테스트 성공
□ 추천 API가 맛 프로필 기반으로 정렬되는지 확인
□ 메뉴 매칭률(%) 계산 로직 정상 동작
□ Qdrant 벡터DB 연동 확인 (추천 시스템)
□ 서버 안정적으로 구동 중
```

### 프론트엔드 체크
```
□ .env 파일에 백엔드 URL 설정
   EXPO_PUBLIC_API_URL=http://백엔드주소:포트
   EXPO_PUBLIC_USE_MOCK=false

□ 앱 빌드 및 실행 테스트
□ 로그인 → 홈 → 레스토랑 상세 플로우 확인
□ 장바구니 → 결제 플로우 확인
```

### 시연 당일 체크
```
□ 와이파이/네트워크 상태 확인
□ 백엔드 서버 구동 상태 확인
□ 앱 미리 로그아웃 상태로 준비 (온보딩 보여주기)
□ 화면 밝기/글씨 크기 적절한지 확인
□ 시연 기기 배터리 충전
□ 백업 영상 준비 (혹시 모를 오류 대비)
```
