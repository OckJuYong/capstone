## Project Background
- ### 필요성
  - 리뷰 대중화로 인한 리뷰 이벤트와 인위적인 서비스로 신뢰도 저하 (2024년 쿠팡 리뷰 조작 공정거래법 위반 혐의 고발)
  - 65.2%가 '리뷰 이벤트' 참여를 위해 후기를 작성했다고 응답하며, 리뷰 신뢰도 감소
  - 코로나 이후에도 지속적으로 상승하는 소상공인 폐업률 (2023년 음식점업 폐업률 16.2%)
  - 분명 맛있는데도 알려지지 않은 맛집들의 홍보 및 노출 부족 문제
  - 음식점 선택 시 가장 중요한 요소는 '맛'이지만, 현재 리뷰 시스템으로는 개인의 입맛을 정확히 반영하기 어려움
- ### 기존 해결책의 문제점
  - 배달앱 및 맛집 플랫폼들의 리뷰 조작 문제로 인한 신뢰성 저하
  - 일반 별점 리뷰 시스템은 개인의 맛 취향을 반영하지 못함
  - 광고 비용을 지불하지 않는 소상공인의 노출 기회 부족
  - 단골 고객 관리 시스템의 부재
  - 유통기한 임박 식재료 처리 방안 미흡
  
## System Design
  - ### System Requirements
    - Frontend: HTML5, CSS3, JavaScript, React, React Native
    - Backend: Spring Boot, Spring Security, RabbitMQ, Kafka, Nginx, Docker
    - Database: MySQL, Qdrant (Vector DB), MongoDB
    - Open API: Naver Map API, Kakao Login API, MoviePy, Pixabay
    - AI/ML: GPT-4 (문맥 기반 데이터 분석 및 맛 표현 매핑), Qdrant 벡터 유사도 기반 추천 시스템
    - 데이터 수집: 네이버 지도 메뉴 크롤링, 네이버 블로그 리뷰 크롤링, KSS(Korean Sentence Splitter)를 활용한 문장 단위 분석
    
## Case Study
  - ### Description
    - 데이터 수집 및 전처리: 네이버 지도와 블로그에서 음식점 메뉴 및 리뷰 데이터 수집. GPT-4를 활용하여 문맥을 분석하고 메뉴와 맛 표현을 매핑
    - 벡터 생성 로직: 수집된 리뷰 데이터를 KSS로 문장 단위로 분할 후, GPT를 통해 6차원 맛 벡터(매운맛, 단맛, 신맛, 느끼한맛, 짠맛, 감칠맛)로 정량화
    - 추천 시스템 평가: 실제 사용자 대상 Top-K 평가 결과, 추천한 5개 메뉴 중 4개를 선호하는 비율 66%, 3개를 선호하는 비율 33% 달성
  
  
## Conclusion
  - ### 신뢰성
맛 표현 기반 정량화된 추천으로 리뷰 조작 문제 해결
러프집합 이론을 활용한 맛 표현 신뢰도 분류
벡터 유사도 기반 추천으로 지속적 학습을 통한 정확도 향상

  - ### 편리성
개인 입맛 취향에 맞는 메뉴 추천으로 선택의 편의성 증대
나만의 맛 프로필 생성 및 관리

  - ### 소상공인 지원
광고비 없이도 평등하게 가게가 노출될 수 있는 기회 제공
재고 부담 감소 및 손실 최소화로 수익 극대화
신규 고객 유입 및 단골 확보로 안정적인 매출 구조 형성
QR 코드 기반 단골 관리 시스템으로 재방문율 증가

  - ### 소비자 혜택
사용자 취향에 맞춘 메뉴 추천으로 만족도 향상
할인된 가격으로 음식 제공받아 고물가 시대 경제적 부담 감소
지역 상권 및 골목 상권 활성화 기여
